Git Commands for Coding in the Command Line

  • To clone a repository, copy repository address from GitHub or 
    $ git clone git@github.com:USER-NAME/REPOSITORY-NAME.git
  • To push updates to the GitHub website/repository, use 
    $ git push
  • To push to a branch other than "main", use 
    $ git push origin [BRANCH]
  • To add all modified files to the staging area 
    $ git add .
  • To add a specific file to the staging area, 
    $ git add [FILENAME]
  • To commit changes in modified files, 
    $ git commit -m "A message describing what you have done to make this snapshot different"
  • To check the current status of a project, 
    $ git status
  • To view a project log, 
    git log

NOTE: A best practice to keep in mind is to make atomic commits, meaning to focus on changing only one feature or task at a time.
